# Seller-apis: Автообновление цен и остатков на Ozon и Яндекс Маркете

Скрипты помогают автоматически обновлять информацию о товарах (цены и остатки) на ваших витринах в Ozon и Яндекс Маркете. 
Всё, что нужно сделать — один раз настроить подключение, а дальше скрипт будет работать сам.

## Что вы получите?

- **Актуальные остатки**. Покупатели видят реальное количество товаров — то, что есть на складе.
- **Правильные цены**. Вы меняете цену в одном месте (например, в файле поставщика) — она автоматически обновляется на маркетплейсах.
- **Экономию времени**. Не нужно вручную править карточки товаров на двух площадках.
- **Меньше ошибок**. Скрипт исключает опечатки и случайные пропуски при обновлении.

## Как это работает (простыми словами)

1. Скрипт скачивает файл с актуальными остатками и ценами (например, с сайта поставщика или из вашей учётной системы).
2. Сверяет артикулы товаров из файла с теми, что уже загружены в Ozon и Яндекс Маркет.
3. Отправляет на маркетплейсы только изменившиеся данные — новые цены и количество товара.
4. Всё происходит автоматически. Вы просто запускаете скрипт.

## Состав проекта и как запустить

В проекте два независимых скрипта — для каждой площадки свой.

### 1. `seller.py` — для Ozon

**Что делает:**
- Загружает остатки и цены из внешнего источника.
- Обновляет эту информацию в вашем магазине Ozon.

**Как запустить:**
1. Подготовьте файл с переменными окружения (см. ниже).
2. Выполните команду:
```bash
python seller.py
```
### 2. `market.py` — для Яндекс Маркета

**Что делает:**
- Загружает остатки и цены
- Поддерживает два типа складов:
  - **FBS** — вы храните товар на своём складе и сами отправляете заказы
  - **DBS** — товар отправляется напрямую от поставщика (дропшиппинг)

**Как запустить:**
1. Подготовьте файл с переменными окружения (см. ниже)
2. Выполните команду:
```bash
python market.py
```
## Настройка (всё, что нужно знать)

Часть данных скрипт берет из переменных окружения. Это безопасно и удобно.  
Просто создайте файл `.env` в папке со скриптами и пропишите там нужные ключи.

### Для Ozon (`seller.py`):

```ini
SELLER_TOKEN = ваш_токен_продавца_озон
CLIENT_ID = ваш_id_клиента_озон
```
### Для Яндекс Маркета (`market.py`):

```ini
MARKET_TOKEN = ваш_токен_яндекс_маркета
FBS_ID = идентификатор_кампании_FBS
DBS_ID = идентификатор_кампании_DBS
WAREHOUSE_FBS_ID = идентификатор_склада_FBS
WAREHOUSE_DBS_ID = идентификатор_склада_DBS
```
### Где взять эти данные?
- Токены и ID выдаются в личных кабинетах продавца на Ozon и Яндекс Маркете.
- Данные об остатках скрипт скачивает сам по ссылке из функции `download_stock()`
(обычно это архив `ostatki.zip` с сайта поставщика).
